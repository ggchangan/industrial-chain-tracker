# 市场验证功能规划

## 功能目标

把产业研究逻辑、公司映射、事件证据和股价走势连接起来，形成：

```text
产业信号 → 研究逻辑 → 受益公司 → 市场表现 → 后续验证或证伪
```

该功能用于辅助观察市场如何反映产业逻辑，不把股价上涨直接等同于逻辑正确，也不构成投资建议。

## 核心使用场景

1. 在产业链、研究链或逻辑卡片中点击公司，查看对应股票行情。
2. 拖动时间窗口，观察公司在某个研究阶段的区间表现。
3. 将涨价、订单、扩产、政策、财报和产品验证等事件标注在走势图上。
4. 对比个股、产业链内其他公司、行业指数和宽基指数的同期表现。
5. 回看某轮行情对应的原始证据，判断逻辑是得到验证、尚待验证还是已经证伪。

## 第一阶段：公司卡中的走势验证入口

低成本完成公司与行情页面的连接：

- 为公司建立统一证券代码字段。
- 区分上交所、深交所、北交所、港股和美股代码。
- 在公司名称旁增加“查看行情”入口。
- A 股优先跳转新浪财经对应的行情或 K 线页面。
- 公司卡展示证券代码、关联研究逻辑和研究日期，作为打开 K 线后的对照节点。
- 外部页面使用新窗口打开，不影响当前研究阅读位置。
- 对未上市公司和无法匹配的公司隐藏行情入口。

这一阶段不依赖项目自行维护行情数据，也不调用未公开行情接口。新浪页面只作为外部
行情工具，不在项目内嵌入。

## 第二阶段：项目内区间表现

在项目内增加“市场验证”模块：

- 可拖动起止日期。
- 显示区间涨跌幅。
- 显示区间最大涨幅、最大回撤和振幅。
- 提供近一周、近一月、近三月、近一年等快捷区间。
- 同期对比沪深 300、中证 1000 或对应行业指数。
- 计算相对基准的超额收益。
- 支持产业链内多家公司走势归一化对比。

建议用复权收盘价计算区间收益，并明确前复权、后复权或不复权口径。

## 第三阶段：逻辑事件叠加

把已有研究数据映射到行情时间轴：

- 动态追踪事件标记。
- 逻辑卡片创建、更新和失效时间。
- 公告、财报、调研、涨价函、订单和政策节点。
- 点击事件后展示摘要、证据强度、来源链接和关联产业链。
- 支持标记“逻辑提出”“首次验证”“持续强化”“出现反证”“逻辑失效”。

最终可以回答：

- 市场是在事件之前还是之后开始反应？
- 哪些公司最先反应，哪些公司存在滞后？
- 行情来自产业逻辑，还是来自大盘和市场风格？
- 逻辑持续时间与股价表现是否匹配？

## 数据结构建议

公司对象至少补充：

```json
{
  "name": "示例公司",
  "ticker": "600000",
  "exchange": "SSE",
  "market": "CN",
  "quoteProvider": "sina",
  "quoteUrl": "https://finance.sina.com.cn/..."
}
```

市场事件建议与现有动态追踪、逻辑卡片保持关联，而不是另建一套互不相通的数据：

```json
{
  "date": "YYYY-MM-DD",
  "type": "订单 / 涨价 / 财报 / 政策 / 反证",
  "title": "事件摘要",
  "chainId": "产业链 ID",
  "companyIds": ["公司 ID"],
  "logicCardIds": ["逻辑卡片 ID"],
  "sourceUrl": "原始来源",
  "confidence": "待核验 / 已核验 / 高置信"
}
```

## 技术与合规注意事项

- 实施前核对新浪财经页面跳转规则、嵌入限制、接口稳定性和使用条款。
- 第一阶段优先采用外链，不依赖非公开接口或页面抓取。
- 项目内绘制 K 线需要选择来源稳定且许可清晰的行情数据。
- 浏览器直接请求第三方行情可能受到跨域限制，必要时由服务端统一获取和缓存。
- 行情数据应记录更新时间、复权方式、交易日历和数据来源。
- 页面应明确区分事实、研究判断和市场表现，避免暗示确定性收益关系。

## 当前优先级

启动第一阶段：先完成 A 股公司卡、证券代码、关联逻辑日期和新浪行情外链。项目内 K 线、
拖动窗口和收益计算仍需选择许可清晰、接口稳定的数据源后实施。
