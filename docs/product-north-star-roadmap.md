# 产品北极星与阶段路线图

## 一句话定义

产业链研究库不是资讯站，也不是行情软件，而是：

```text
帮助用户建立产业链骨架认知，并持续跟踪产业逻辑如何被新资料、新证据和市场表现验证或修正的研究系统。
```

它要同时服务两件事：

- 学习产业链：让用户知道一条产业链由哪些环节组成，价值如何传导，关键公司在哪里。
- 跟踪投资逻辑：让用户知道哪些逻辑正在强化、减弱、出现反证，以及市场是否开始交易这些逻辑。

## 用户价值

目标用户不是只想看新闻的人，而是希望系统理解产业、形成投资判断的人。

典型问题：

- 这条产业链到底怎么运转？
- 我应该按什么顺序学习它？
- 最近新增的研报、公众号文章、抖音视频或公告改变了哪条产业逻辑？
- 哪个环节正在变紧、变贵、变重要？
- 哪些公司和这条逻辑最相关？
- 市场有没有开始交易这条逻辑？
- 哪些证据会让这条逻辑强化、减弱或失效？

## 产品核心对象

### 1. 产业链骨架

产业链骨架是学习基础，类似 PCB 产业链当前的上中下游结构。

它回答：

- 上游、中游、下游分别是什么？
- 关键材料、设备、工艺、产品和应用在哪里？
- 哪些公司处于哪些位置？
- 价值量、瓶颈、利润弹性和传导关系大致如何？

骨架不需要每天变化。它是用户学习产业链的地图底座。

### 2. 学习路线

学习路线基于产业链骨架生成。

它回答：

```text
一个新用户应该按什么顺序理解这条产业链？
```

示例：

```text
PCB 总骨架
-> CCL 为什么重要
-> 电子布 / 铜箔 / 树脂如何影响 CCL
-> PCB 厂商如何传导成本
-> 相关公司和市场验证
```

学习路线不是另写一套课程，而是把骨架节点按认知顺序重新组织。

### 3. 资料

资料是持续更新的输入，包括：

- 券商 PDF。
- 公众号文章。
- 抖音 / 视频链接。
- 公司公告。
- 财报。
- 调研纪要。
- 价格、订单、扩产、认证、政策等事件资料。

资料不能只作为列表堆积。每份资料都应该尽量挂到：

- 某条产业链。
- 某个骨架节点。
- 某条逻辑。
- 某家公司。
- 某个证据或监控项。

### 4. 逻辑

逻辑是从资料中提炼出的判断。

示例：

```text
MPO 用量提升
-> MT 插芯供需紧张
-> 扩产周期较长
-> 相关公司订单和毛利率可能改善
```

逻辑不是资料摘要。逻辑必须能回答：

- 为什么重要？
- 影响哪个环节？
- 受益或受损公司是谁？
- 需要哪些证据验证？
- 什么情况会反证？

### 5. 证据

证据用于验证逻辑。

证据类型包括：

- 价格变化。
- 订单或中标。
- 扩产、投产、交期。
- 客户认证。
- 财报收入、毛利率、库存。
- 行业数据。
- 公司公告。
- 产业调研。

证据应有状态：

```text
待核验 / 已核验 / 强化 / 稳定 / 减弱 / 反证 / 失效
```

### 6. 公司与市场表现

公司是产业逻辑和投资观察之间的桥。

当前的 K 线验证逻辑方向是正确的：项目不直接荐股，而是帮助用户观察市场是否在交易某条产业逻辑。

公司模块应回答：

- 这家公司处在产业链哪个环节？
- 它和哪条逻辑相关？
- 这条逻辑提出或强化的时间是什么？
- 市场表现是否同步或滞后？
- 后续还需要哪些产业证据验证？

## 我们与竞品的区别

### 与 AICardMap

AICardMap 更像 AI 硬件卡口百科和会员数据库。

值得学习：

- 固定导航壳。
- 环节原子化。
- 会员锁定。
- 意见反馈。
- 情报雷达。
- 更新记录。

不能照搬：

- 我们不是静态卡口百科。
- 我们的核心不是“有多少个节点可以看”，而是“逻辑如何随资料和证据持续变化”。

我们的差异：

```text
产业链骨架 + 资料时间链 + 逻辑地图 + 证据雷达 + 市场验证
```

### 与研报平台

研报平台解决“找到资料”。

我们要解决：

```text
资料进入产业链之后，改变了哪条逻辑？
```

### 与行情软件

行情软件解决“价格怎么走”。

我们要解决：

```text
价格背后市场正在交易哪条产业逻辑？
```

### 与普通资讯流

资讯流解决“发生了什么”。

我们要解决：

```text
这件事对产业链判断意味着什么？
```

## 核心产品形态

### 产业链地图

产业链地图负责结构。

它展示：

- 上中下游。
- 关键环节。
- 关键公司。
- 价值传导。
- 瓶颈环节。
- 与逻辑节点的连接。

产业链地图回答：

```text
结构在哪里？
```

### 逻辑地图

逻辑地图负责判断。

它展示：

- 基准逻辑。
- 最新研究逻辑。
- 逻辑之间的因果链。
- 逻辑影响的骨架节点。
- 关联公司。
- 支撑证据。
- 反证条件。
- 当前状态。

逻辑地图回答：

```text
判断如何形成，如何变化？
```

### 资料时间链

资料时间链负责记录输入。

它展示：

- 每份资料的日期、来源、类型和摘要。
- 它挂到哪个骨架节点。
- 它影响哪条逻辑。
- 它提供了什么证据。

资料时间链回答：

```text
最近新增了什么资料？
```

### 逻辑雷达

逻辑雷达负责快速跟踪变化。

它不应该只是新闻列表，而应按逻辑组织：

- 今日强化的逻辑。
- 今日减弱的逻辑。
- 出现反证的逻辑。
- 待人工核验的证据。
- 用户订阅产业链的关键变化。

逻辑雷达回答：

```text
哪些判断正在变化？
```

### 市场验证

市场验证负责连接公司和 K 线。

它展示：

- 相关公司。
- 证券代码。
- 关联逻辑。
- 逻辑提出或强化日期。
- 外部 K 线入口。
- 后续可扩展为区间涨跌、事件叠加和多公司对比。

市场验证回答：

```text
市场有没有开始交易这条逻辑？
```

## 第一阶段：样板产业链工作台

第一阶段不要做全量大重构。先选择一条样板产业链做透。

候选：

- PCB：骨架清晰，适合做学习路线。
- 光模块：已有 MPO 研究包，适合做资料时间链、逻辑地图和市场验证。

建议优先选光模块，因为它已有“新增研究主题 MPO”的真实样例。

### 第一阶段目标

把单条产业链从文章页升级为研究工作台：

```text
当前重点
产业链骨架
学习路线
资料时间链
逻辑地图
证据 / 跟踪验证
公司与市场验证
原文与资料档案
```

### 第一阶段只做什么

1. 增加“学习路线”视图  
   基于现有 `chain.chain` 骨架生成，不新增复杂数据模型。

2. 增加“资料时间链”视图  
   把现有 `sources`、`updates`、`researchPackages` 统一展示。

3. 增加“逻辑地图雏形”  
   先基于现有 `logic` 和 `logicTracks.coreInsights` 渲染。

4. 强化“证据如何影响逻辑”的展示  
   每条资料或动态尽量显示它关联的逻辑、环节或公司。

5. 保留原文阅读，但降级为深度入口  
   原文仍是证据底座，不再是唯一主阅读路径。

### 第一阶段暂时不做什么

- 不做复杂会员系统。
- 不做完整自动采集。
- 不做项目内 K 线。
- 不做 AI 自动提炼全流程。
- 不做所有产业链的一次性重构。

## 第二阶段：雷达、订阅与反馈

在样板产业链跑通后，再做产品化运营能力。

### 登录

登录用户可：

- 收藏产业链。
- 订阅逻辑更新。
- 保存阅读历史。
- 提交反馈。

### 意见建议

反馈入口支持：

- 资料错误。
- 公司映射错误。
- 逻辑异议。
- 环节遗漏。
- 希望新增研究主题。

反馈需要自动带上当前页面上下文。

### 逻辑雷达

雷达页面支持：

- 按产业链筛选。
- 按逻辑状态筛选。
- 按证据类型筛选。
- 显示待核验事项。
- 显示订阅产业链的变化。

## 第三阶段：会员与高级能力

会员体系可以做，但要围绕深度研究价值，而不是简单锁数量。

可选权限：

- 完整产业链。
- 完整逻辑地图。
- 历史逻辑变更。
- 雷达订阅。
- 高级资料档案。
- 市场验证增强。

建议保留一个完整公开样板，让用户理解产品价值。

## 近期执行建议

### Step 1：选样板

从光模块开始。

原因：

- 已有 MPO 资料包。
- 逻辑、公司、监控、K 线验证都有雏形。
- 能很好展示“资料进入后如何改变产业链逻辑”。

### Step 2：补设计文档

为光模块写一份页面信息结构草图：

```text
光模块产业链工作台
├── 当前重点
├── 产业链骨架
├── 学习路线
├── 最新资料时间链
├── 逻辑地图
├── 跟踪验证
├── 公司与市场验证
└── 原文与资料
```

样板工作台方案见 `docs/optical-module-workbench-prototype.md`。这份文档同时预留了后续 CPO 抖音短视频资料的录入模板和展示路径。

### Step 3：低成本实现

先用现有数据拼视图，不急着改后台数据结构。

### Step 4：导入更多资料

围绕光模块继续导入：

- 券商 PDF。
- 公众号文章。
- 抖音视频链接。
- 公司公告。

每份资料都要尽量提炼：

```text
影响哪条逻辑？
支撑什么证据？
关联哪个骨架节点？
关联哪些公司？
```

## 产品原则

1. 先骨架，后资料，再逻辑，最后市场验证。
2. 资料不是内容堆积，资料必须服务逻辑更新。
3. 每次更新都要回答：它改变了哪条逻辑？
4. 产业链地图讲结构，逻辑地图讲判断，雷达讲变化。
5. 工具要帮助学习，也要帮助形成投资观察，但不做荐股。
6. 先把一条样板产业链做透，再复制到其他产业链。
