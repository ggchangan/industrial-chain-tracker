# 多端产品路线

## 目标架构

所有客户端共用同一内容服务：

```text
Git 内容仓库
  ↓ 同步与校验
内容 API / 搜索 / 正文渲染
  ├─ 网页版
  ├─ 微信小程序
  ├─ iOS App
  └─ Android App
```

维护者登录和未来的写入 API 与公开阅读 API 分离。真实密码、会话密钥和发布凭证只保存在服务器环境变量或密钥服务中。

## 第一阶段：公共内容服务

已实现：

- 完整资料库与产业链列表 API
- 单条产业链与 Markdown/HTML 正文 API
- 服务端跨产业链搜索
- API 优先、静态数据回退的网页版
- 维护者密码登录、签名 Cookie、登录限流
- 维护页面和维护说明的服务端访问控制

## 第二阶段：微信小程序 MVP

已建立 `apps/mobile` uni-app 工程并实现：

- 产业链列表
- 跨产业链搜索
- 产业链详情与图谱预览
- 核心逻辑、专属追踪和最新动态
- 服务端渲染的正式文章

上线前待办：

- 申请或确认微信小程序 AppID
- 配置已备案的 HTTPS API 域名
- 在微信公众平台加入 request 合法域名
- 在真机检查长表格、图片预览、分享卡片和弱网体验
- 补充隐私政策、用户协议和小程序类目
- 跟随 DCloud 官方工具链升级复查 `npm audit`；当前告警来自构建依赖，不随小程序产物部署

## 第三阶段：阅读体验增强

- 收藏产业链、公司和关键词
- 阅读历史与目录位置恢复
- 动态更新订阅
- 公司交叉图谱
- 搜索结果精准跳转到正文段落
- 分享卡片和产业链二维码

匿名收藏可先保存在本地；需要跨设备同步时再引入微信登录和用户数据库。

## 第四阶段：iOS 与 Android

继续使用 `apps/mobile` 业务代码，按平台补充：

- Apple/Android 登录与推送
- 深色模式和系统字体适配
- 离线缓存与后台更新
- App Store、各安卓市场的隐私和审核材料

## 第五阶段：在线内容管理

当前维护仍采用 Git 工作流。正式在线后台建议增加：

- 产业链草稿、预览与发布
- Markdown 编辑和图片上传
- 动态追踪录入
- 版本历史和回滚
- 操作审计
- 角色权限

写入 API 不与公开 API 使用同一权限域，发布动作必须保留版本记录。
