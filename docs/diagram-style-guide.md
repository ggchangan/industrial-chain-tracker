# 产业链图谱视觉规范

本规范用于公开网页、小程序预览图和内容平台长图，目标是让新增产业链保持统一、清晰和可识别。

## 标准母版

- 视觉参考：`diagram/mlcc-industry-chain/mlcc-industry-chain-map.svg`
- 推荐 SVG 画布：`1600 × 1040` 或 `1680 × 1120`，根据节点密度小幅调整
- PNG 导出：宽度至少 `4000px`，透明度和字体渲染使用 8-bit RGBA
- 网页展示：优先使用 SVG，PNG 仅作为兼容回退和小程序预览资源
- 页面比例：约 `1.5:1`，不为塞入更多内容随意改变画布比例
- 安全边距：左右至少 `54px`，顶部至少 `46px`，底部至少 `32px`

## 固定结构

1. 顶部标题区：产业链名称和单行核心传导路径。
2. 主图谱区：默认上游、中游、下游三大虚线区域；复杂产业链可在区域内拆分子环节。
3. 节点卡片：使用箭头表达真实供给、制造、应用和需求反向拉动关系，不排列成普通信息看板。
4. 底部研究主线：使用四个并列逻辑卡，概括需求、瓶颈、兑现和风险验证。
5. 页脚：`产业链研究库 · 产业链简称 · YYYY.MM.DD`。

## 字体层级

- 标题：约 `32px / 800`
- 副标题：约 `15px`
- 区域标题：约 `18px / 800`
- 节点标题：约 `15px / 800`
- 公司与节点说明：约 `11px`
- 注释与页脚：约 `10-14px`
- 字体栈：`Noto Sans SC, PingFang SC, Microsoft YaHei, sans-serif`
- 字距固定为 `0`，禁止通过缩小字距强行塞入内容

较大画布可按相同比例放大字号，但标题、区域标题和节点标题的层级比例必须保持一致。

## 色彩系统

- 背景：`#0f172a`
- 网格：`#1e293b`
- 主文字：`#f8fafc`
- 次文字：`#94a3b8`
- 青色：`#22d3ee`，用于上游、基础材料和主传导箭头
- 绿色：`#34d399`，用于制造、量产和应用兑现
- 黄色：`#fbbf24`，用于定价、供需和关键瓶颈
- 蓝色：`#60a5fa`，用于设备、平台和系统支撑
- 紫色：`#a78bfa`，用于高端工艺和技术代际
- 红色：`#fb7185`，仅用于风险、紧缺和重点变化

单张图必须同时使用多个功能色，不能被一种色相主导。颜色表达产业角色，不表达股票涨跌或投资评级。

## 内容密度

- 三段母版宽度根据内容分配，上游和下游可宽、中游可窄，保留足够箭头空间
- 节点卡片圆角约 `10-16px`，外层虚线区域圆角约 `18px`
- 每张卡片最多三层：标题、公司、技术说明
- 公司过多时保留代表性公司，完整名单放到网页产业链骨架
- 禁止使用星级、排名、买入、推荐、核心受益等投资导向标记
- 事件、价格和机构观点不直接写成确定事实，需注明日期和核验状态

## 交付检查

- SVG 中的中文字体和特殊字符显示正常
- 标题、阶段、公司名称没有溢出或遮挡
- 箭头方向与产业传导关系一致
- 网页在 1280px 桌面宽度下文字清晰
- PNG 在微信预览中可读，不能由低分辨率图片二次放大生成
- SVG 与 PNG 内容一致，文件路径同时登记到 `assets/data.js`
