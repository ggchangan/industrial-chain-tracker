# 光模块产业链样板工作台方案

## 目标

用光模块产业链做第一条样板，把项目从“产业链文章页”推进到“产业链学习与逻辑跟踪工作台”。

光模块适合作样板，因为它已经具备：

- 清晰的上游、中游、下游骨架。
- 基准逻辑：光进铜退、高速模块放量、CPO 价值重构、底层网络升级。
- 跟踪项：800G/1.6T 订单、光芯片瓶颈、CPO 与硅光、薄膜铌酸锂、光纤光缆组网。
- 后续可接入 MPO、CPO、抖音短视频、券商 PDF、公众号文章、公告和调研纪要。

## 产品定位

光模块工作台不只是展示光模块产业链内容，而是回答：

```text
AI 算力需求如何沿着光模块产业链传导？
哪些环节正在变紧、变贵、变重要？
新资料改变了哪条产业逻辑？
哪些公司与这些逻辑相关？
市场有没有开始交易这些逻辑？
```

## 页面结构

建议单条产业链页面按以下顺序组织：

```text
光模块产业链工作台
├── 当前重点
├── 产业链骨架
├── 学习路线
├── 资料时间链
├── 逻辑地图
├── 跟踪验证
├── 公司与市场验证
└── 原文与资料档案
```

## 1. 当前重点

当前重点是用户进入页面后最先看的区域，不超过 3 条。

首版可从现有数据自动拼：

- 最新资料或研究主题。
- 最新逻辑变化。
- 最新待核验信号。

光模块当前可展示：

```text
1. 高速光模块订单：800G / 1.6T 订单、海外云厂 CAPEX、产能扩张。
2. CPO 与硅光：Spectrum-X、CPO 量产、硅光占比、光引擎出货。
3. 光芯片瓶颈：EML、CW 光源、InP 衬底、良率和批量交付。
```

后续如果录入 CPO 抖音更新，当前重点应自动出现：

```text
CPO 相关资料更新：短视频提到的量产节奏、产业链环节或公司线索，进入待核验。
```

## 2. 产业链骨架

产业链骨架是学习基础，不需要每天变化。

当前光模块骨架可以保留为三层：

### 上游：光芯片与材料

- 光芯片：EML、CW 光源、PLC / AWG。
- 薄膜铌酸锂与光学晶体：TFLN 调制器、光学晶体。
- 衬底与陶瓷封装：InP 衬底、陶瓷外壳、陶瓷基板、插芯。
- 光纤预制棒与光缆：光棒、光纤、光缆。

### 中游：光器件与光模块

- 光器件与光引擎：无源光器件、光引擎、OCS / CPO 零部件。
- 高速光模块：800G、1.6T、LPO、硅光模块。

### 下游：智算组网

- CPO 硅光交换机。
- 光纤光缆网络。
- 交换机与云厂。

## 3. 学习路线

学习路线应基于骨架生成，帮助新用户按顺序理解。

建议光模块第一版学习路线：

```text
1. 先理解 AI 算力为什么需要高速互联
2. 理解 800G / 1.6T 光模块如何承接需求
3. 理解光芯片、光器件和封装为什么成为供给瓶颈
4. 理解 CPO / 硅光为什么可能重构价值链
5. 理解光纤光缆和交换机如何决定网络落地
6. 最后看相关公司和 K 线验证
```

每一步都应能跳到：

- 对应骨架节点。
- 对应逻辑卡。
- 对应资料。
- 对应公司。

## 4. 资料时间链

资料时间链负责记录所有新增输入。

资料类型包括：

- 基准原文。
- 券商 PDF。
- 公众号文章。
- 抖音 / 短视频链接。
- 公司公告。
- 财报。
- 调研纪要。

时间链不应只是按时间列资料。每条资料都必须尽量补齐：

```text
资料来源
资料类型
影响的骨架节点
影响的逻辑
提供的证据
相关公司
当前状态：待核验 / 已归档 / 已提炼逻辑 / 已进入雷达
```

### CPO 抖音更新的录入模板

用户后续提供 CPO 抖音链接时，可以按这个结构录入：

```json
{
  "date": "YYYY-MM-DD",
  "type": "short-video",
  "platform": "Douyin",
  "title": "视频标题或一句话摘要",
  "originalUrl": "抖音链接",
  "summary": "视频主要观点，不直接当作事实结论。",
  "segment": "CPO与硅光",
  "companies": ["天孚通信", "光库科技", "腾景科技"],
  "tags": ["CPO", "硅光", "光引擎", "Spectrum-X"],
  "status": "draft"
}
```

入库后应生成一条时间链记录：

```text
YYYY-MM-DD · 短视频 · CPO 与硅光
视频提出 CPO 量产或产业链环节变化线索，当前作为待核验证据。
影响逻辑：CPO 价值重构
关联骨架：下游 CPO 硅光交换机 / 中游光器件与光引擎 / 上游薄膜铌酸锂
状态：待核验
```

## 5. 逻辑地图

逻辑地图是光模块样板的核心。

首版可以先用现有逻辑渲染：

```text
AI 算力需求扩大
  -> 光进铜退
    -> 高速模块放量
      -> 光芯片 / 光器件 / 封装瓶颈
        -> 公司业绩兑现

AI 集群网络升级
  -> CPO 与硅光
    -> 光引擎、硅光、TFLN、精密器件价值重构
      -> 量产节奏和客户验证决定逻辑强弱

智算网络底座升级
  -> 多芯光缆商用 / 光纤涨价
    -> 光纤光缆与组网公司受益
```

每个逻辑节点至少显示：

- 逻辑标题。
- 当前状态。
- 影响骨架节点。
- 相关资料。
- 支撑证据。
- 待观察信号。
- 相关公司。
- 原文入口。

### CPO 逻辑节点

CPO 更新应主要挂在“CPO 价值重构”节点下：

```text
CPO 价值重构
├── 触发因素：AI 集群网络带宽和功耗约束
├── 产业链影响：可插拔模块 -> 光引擎 / 硅光 / TFLN / 高精密器件
├── 待验证信号：Spectrum-X、CPO 量产、硅光占比、光引擎出货
├── 相关公司：天孚通信、光库科技、腾景科技、福晶科技
└── 反证条件：量产节奏推迟、成本不经济、可靠性验证不及预期
```

抖音短视频进入后，不应直接把逻辑状态改成“强化”。更合理的流程：

```text
短视频资料入库
-> 生成待核验证据
-> 维护者确认来源和事实
-> 若与公告、研报、订单或客户验证形成交叉印证
-> 逻辑状态改为强化
```

## 6. 跟踪验证

跟踪验证负责把逻辑变成可观察的问题。

光模块当前跟踪项：

- 高速光模块订单。
- 光芯片瓶颈。
- CPO 与硅光。
- 薄膜铌酸锂。
- 光纤光缆组网。

建议展示方式：

```text
监控项
├── 为什么重要
├── 要看哪些信号
├── 当前证据
├── 最近资料
├── 当前判断
└── 下一次需要核验什么
```

CPO 抖音更新进入后，应出现在：

- 资料时间链。
- CPO 价值重构逻辑节点。
- CPO 与硅光跟踪项的最近资料。
- 逻辑雷达的“待核验”区域。

## 7. 公司与市场验证

公司与市场验证回答：

```text
这条逻辑映射到哪些公司，市场有没有开始交易？
```

光模块首版公司分组：

### 高速光模块

- 中际旭创。
- 新易盛。
- 剑桥科技。
- 华工科技。

### 光芯片

- 源杰科技。
- 长光华芯。
- 仕佳光子。

### CPO / 硅光 / 光引擎

- 天孚通信。
- 光库科技。
- 腾景科技。
- 福晶科技。

### 上游材料与封装

- 云南锗业。
- 中瓷电子。
- 三环集团。
- 国瓷材料。

### 光纤光缆与组网

- 长飞光纤。
- 亨通光电。
- 中兴通讯。

公司卡需要显示：

- 所属骨架节点。
- 关联逻辑。
- 最近资料日期。
- 最新证据状态。
- K 线入口。

## 8. 原文与资料档案

原文是证据底座，但不是第一阅读入口。

建议：

- 工作台首屏不直接铺长文。
- 原文入口保留在底部或侧栏。
- 资料档案按类型筛选：原文、PDF、公众号、短视频、公告、调研。
- 每份资料都显示是否已经提炼逻辑、是否进入雷达。

## 第一版实现范围

不改底层数据模型，先用现有字段拼出样板视图：

- `chain.chain` -> 产业链骨架、学习路线。
- `chain.logic` -> 基准逻辑地图。
- `chain.logicTracks` -> 最新研究逻辑。
- `chain.updates` -> 资料时间链的一部分。
- `chain.sources` -> 资料档案。
- `chain.trackingProfile.metrics` -> 跟踪验证。
- `chain.watchlist` -> 待观察信号。
- `companySecurities` -> 公司与 K 线入口。

## 后台录入要求

后续录入资料时，维护台应尽量提示维护者补充：

```text
这份资料属于哪个骨架节点？
它影响哪条逻辑？
它提供了什么证据？
它关联哪些公司？
它是强化、减弱、反证，还是仅待核验？
```

这比单纯填写“标题、链接、摘要”更重要。

## 验收标准

光模块样板完成后，用户应能：

1. 通过骨架快速理解光模块产业链结构。
2. 按学习路线从新手视角理解光模块。
3. 看到 MPO、CPO 等新增资料在时间链中的位置。
4. 明白新增资料影响了哪条逻辑。
5. 在逻辑地图上看到 CPO、光芯片、高速模块等判断如何连接。
6. 在跟踪验证中看到哪些证据仍待确认。
7. 点击公司进入 K 线验证入口，观察市场是否交易相关逻辑。
